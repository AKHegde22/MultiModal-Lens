Why We Built This

TransformerLens changed how we debug language models. It gave researchers a way to look inside transformers: caching activations, tracing attention heads, running causal interventions with a few lines of code. But when the field moved to vision-language models, that toolkit hit a wall.

"Vision-language models expose rich internals: attentions, hidden states, embeddings. But interpretability tooling remains architecture-specific and difficult to operationalize." MultimodalLens Research Paper

Every VLM family handles images differently. CLIP uses dual encoders. BLIP-2 routes through a Q-Former. LLaVA interleaves image tokens into a decoder. Each requires its own extraction logic, its own attention surgery, its own faithfulness testing. There was no unified tool.

MultimodalLens fills that gap. A single adapterized framework that normalizes the chaos of 20+ VLM architectures into one debugging interface, with built-in faithfulness checks to make sure your heatmaps aren't lying to you.

One Interface, Every Architecture

All model-specific logic is isolated behind a single ModelAdapter interface. The UI, CLI, and evaluation layers stay architecture-agnostic.

Adapterized Abstraction

Four canonical adapters each implement load(), prepare(), analyze(), score(). The UI and CLI never see model-specific code.

TransformerLens-Style Hooks

Three hook classes built on PyTorch's register_forward_hook. Auto-discovers transformer layers by scanning model.named_modules().

Declarative Configs

New VLM families via MultimodalConfig: define vision tower, projector, language model paths, and image token string. No custom adapter code needed.

What MultimodalLens Can Do

From attention heatmaps to causal path patching. A mechanistic interpretability toolkit, not a visualization dashboard.

Attention Rollout & Overlays

Residual-augmented rollout across all layers: A'_l = normalize(A_l + I), then R = A'_L * A'_{L-1} * ... * A'_1. Reshaped to patch grids and overlaid as heatmaps on input images.

Token-Patch Alignment

Cross-modal cosine similarity matrices between text tokens and visual patches in shared embedding space. S_ij = cos(t_i, v_j) with per-token saliency.

Multimodal Logit Lens

Decode intermediate hidden states through lm_head layer-by-layer. Three-stage vision logit lens for pre-projector, post-projector, and language layer analysis.

Cross-Modal Activation Patching

Causal tracing by swapping activations between source and target images at a selected layer. If replacing visual-token positions changes the output, that layer carries causal evidence.

Grounding Head Discovery

Identify attention heads responsible for visual grounding via ablation sensitivity. Ranks heads by |delta_visual_mass| * (1 + max(score_drop, 0)).

Faithfulness Diagnostics

Attention is not explanation. Deletion/insertion curves, counterfactual masking, and attention-gradient Spearman correlation verify whether heatmaps reflect causal importance.

Direct Logit Attribution

Decompose target token logit into per-head, per-MLP, and embedding contributions. Interactive waterfall visualization of exactly which components push the prediction.

Causal Path Patching

Edge-level intervention analysis between sender and receiver components across Q/K/V channels. Maps the full causal graph of information flow through the model.

How It Works

From raw image and text to causal insights.

Input

PIL image and prompt string. AutoProcessor tokenizes text and preprocesses images into model-ready tensors.

Route

Family resolution picks the right adapter. CLIP extracts dual towers, BLIP-2 routes through Q-Former, LLaVA segments decoder states.

Forward

Model runs with attentions and hidden states enabled. Forward hooks capture per-layer activations without modifying the model.

Analyze

Attention rollout, alignment, gradients, faithfulness probes, and mechanistic analysis run on cached activations.

Output

Unified AnalysisResult: heatmaps, matrices, scores, faithfulness metrics. Ready for UI display or batch export.

quickstart.py 
from multimodallens import HookedVLM
from PIL import Image

# Load any supported VLM in one line
vlm = HookedVLM.from_pretrained("openai/clip-vit-base-patch32", device="auto")

# Run with cache
result, cache = vlm.run_with_cache(Image.open("cat.jpg"), "a photo of a cat")

# Access activation by layer name
layer_5 = cache["vision_encoder.layers.5"]

# Zero-ablate a layer on the fly
patched = vlm.run_with_hooks(
    image=Image.open("cat.jpg"),
    prompt="a photo of a cat",
    fwd_hooks=[("vision_encoder.layers.5", lambda t: t * 0.0)]
)

The Science Behind the Lens

Three complementary evidence channels. No single explanation primitive.

Structural Signal: Attention

A'_l = normalize(A_l + I)
R = A'_L * A'_{L-1} * ... * A'_1

Residual-augmented rollout propagates attention through all layers while accounting for skip connections. The resulting matrix captures each token's total attention to every image patch across the full network depth.

Representation Signal: Alignment

S_ij = cos(t_i, v_j)
salience_i = max_j S_ij

Token-patch cosine similarity in shared latent space reveals which text tokens align with which visual patches. The alignment matrix and per-token saliency scores expose cross-modal grounding.

Causal Signal: Faithfulness

Deletion: mask top-ranked patches, track delta_score
Insertion: preserve top patches, track score recovery
Agreement: Spearman(attn, gradient)

Attention alone is not a faithful explanation. Deletion/insertion curves, counterfactual drops, and attention-gradient Spearman correlation validate whether attention maps reflect genuine causal importance.

Mechanistic Probes

Hook → Capture → Decode/Project → Analyze

Logit Lens: hidden_state @ lm_head → vocab
Path Patching: sender → receiver edge effects

TransformerLens-style probes: forward-hook activation caching, cross-modal activation patching, multimodal logit lens, grounding head discovery, direct logit attribution, and causal path patching.

The Diagnostic Tuple

D = (A, S, g, F)

For every (image, prompt) pair, MultimodalLens produces A (attention maps), S (alignment matrix), g (global score), and F (faithfulness metrics). Four convergent signals for robust model debugging.
