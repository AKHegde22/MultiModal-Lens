The Origin

Why We Built This

TransformerLens changed how we debug language models. It gave researchers a way to look inside transformers — caching activations, tracing attention heads, and running causal interventions with a few lines of code. But when the field moved to vision-language models, that toolkit hit a wall.

"Vision-language models expose rich internals — attentions, hidden states, embeddings — but interpretability tooling remains architecture-specific and difficult to operationalize."
— MultimodalLens Research Paper

Every VLM family handles images differently. CLIP uses dual encoders. BLIP-2 routes through a Q-Former. LLaVA interleaves image tokens into a decoder. Each requires its own extraction logic, its own attention surgery, its own faithfulness testing. There was no unified tool.

MultimodalLens fills that gap — a single adapterized framework that normalizes the chaos of 20+ VLM architectures into one clean debugging interface, with built-in faithfulness checks to make sure your heatmaps aren't lying to you.

Architecture

Adapterized Model Abstraction

All model-specific logic is isolated behind a single ModelAdapter interface. The UI, CLI, and evaluation layers stay completely architecture-agnostic.

⚡ Adapterized Abstraction

Four canonical adapters — CLIPAdapter, BLIP2Adapter, LlavaAdapter, GenericVLMAdapter — each implement load(), prepare(), analyze(), and score(). The UI and CLI never see model-specific code.

🔗 TransformerLens-Style Hooks

Three hook classes — ForwardHookCache, ForwardLayerPatcher, ForwardInputPatcher — built on PyTorch's register_forward_hook. Auto-discovers transformer layer paths by scanning model.named_modules().

📦 Declarative Configs

New VLM families can be added via MultimodalConfig — define the vision tower path, projector path, language model path, and image token string. No custom adapter code needed for simple architectures.

Capabilities

What MultimodalLens Can Do

From attention heatmaps to causal path patching — a complete mechanistic interpretability toolkit for the multimodal era.

01

Attention Rollout & Overlays

Residual-augmented rollout across all layers: A'_l = normalize(A_l + I), then R = A'_L · A'_{L-1} · ... · A'_1. Reshaped to patch grids and overlaid as heatmaps on input images. Also supports last-layer mean-head views and gradient attribution maps.

02

Token-Patch Alignment

Cross-modal cosine similarity matrices between text tokens and visual patches in shared embedding space. S_ij = cos(t_i, v_j) — shown as token-by-patch heatmaps with per-token saliency scores.

03

Multimodal Logit Lens

Decode intermediate hidden states through lm_head layer-by-layer, yielding a trajectory of what the model "believes" before final decoding. Three-stage vision logit lens for pre-projector, post-projector, and language layer analysis.

04

Cross-Modal Activation Patching

Causal tracing by swapping activations between source and target images at a selected layer. If replacing visual-token positions changes the output score, that layer carries causal visual evidence for text generation.

05

Grounding Head Discovery

Identify specific attention heads responsible for visual grounding via ablation sensitivity analysis. Ranks heads by |Δ_visual_mass| × (1 + max(score_drop, 0)) — the candidates behind image-conditioned reasoning.

06

Faithfulness Diagnostics

Because attention is not explanation. Deletion/insertion curves, counterfactual masking drops, and attention-gradient Spearman correlation verify whether your heatmaps actually reflect causal importance.

07

Direct Logit Attribution

Decompose target token logit into per-head, per-MLP, and embedding contributions. Interactive waterfall visualization showing exactly which components push the model toward its prediction.

08

Causal Path Patching

Edge-level intervention analysis between sender and receiver components across Q/K/V channels. Maps the full causal graph of information flow through the model's attention and MLP layers.

Pipeline

How MultimodalLens Works

From raw image and text to causal insights — a five-stage pipeline.

01 — INPUT

Image + Text

PIL image and prompt string enter the system. AutoProcessor tokenizes text and preprocesses images into model-ready tensors.

02 — ADAPTER

Model-Specific Routing

Family resolution picks the right adapter. CLIP extracts dual towers, BLIP-2 routes through Q-Former, LLaVA segments decoder hidden states.

03 — FORWARD

Hooked Forward Pass

Model runs with output_attentions=True, output_hidden_states=True. Forward hooks capture per-layer activations without modifying the model.

04 — ANALYZE

Multi-Signal Extraction

Attention rollout, token-patch alignment, gradient attribution, faithfulness probes, and mechanistic analysis run in parallel on cached activations.

05 — OUTPUT

Normalized Results

All adapters produce a unified AnalysisResult — heatmaps, matrices, scores, faithfulness metrics, ready for UI display or batch export.

quickstart.py 
from multimodallens import HookedVLM
from PIL import Image

# Load any supported VLM in one line
vlm = HookedVLM.from_pretrained("openai/clip-vit-base-patch32", device="auto")

# Run with cache — get logits, scores, and all internal layer activations
result, cache = vlm.run_with_cache(Image.open("cat.jpg"), "a photo of a cat")

# Access any activation by layer name
layer_5 = cache["vision_encoder.layers.5"]

# Zero-ablate a layer on the fly
patched = vlm.run_with_hooks(
    image=Image.open("cat.jpg"),
    prompt="a photo of a cat",
    fwd_hooks=[("vision_encoder.layers.5", lambda t: t * 0.0)]
)

Methodology

The Science Behind the Lens

MultimodalLens combines three complementary evidence channels to avoid over-reliance on any single explanation primitive.

🔍 Structural Signal — Attention

A'l = normalize(Al + I)
R = A'L · A'L-1 · ... · A'1

Residual-augmented rollout propagates attention through all layers while accounting for skip connections. The resulting matrix R captures each token's total attention to every image patch across the full depth of the network.

🧬 Representation Signal — Alignment

Sij = cos(ti, vj)
saliencei = maxj Sij

Token-patch cosine similarity in shared latent space reveals which text tokens are most aligned with which visual patches. The alignment matrix S and per-token saliency scores expose cross-modal grounding.

🧪 Causal Signal — Faithfulness

Deletion: mask top-ranked patches, track Δscore
Insertion: preserve top patches, track score recovery
Counterfactual: single-step top-k% masking effect
Agreement: Spearman(attn, gradient)

Attention alone is not a faithful explanation. Deletion/insertion curves, counterfactual drops, and attention-gradient Spearman correlation validate whether attention maps reflect genuine causal importance.

🔬 Mechanistic Probes

Forward Hook → Capture → Decode/Project → Analyze

Logit Lens: hidden_state @ lm_head → vocab
Path Patching: sender → receiver edge effects
DLA: per-head + per-MLP logit decomposition

TransformerLens-style probes — forward-hook activation caching, cross-modal activation patching, multimodal logit lens, grounding head discovery, direct logit attribution, and causal path patching.

The Diagnostic Tuple

D = (A, S, g, F)

For every (image, prompt) pair, MultimodalLens produces A (attention maps), S (alignment matrix), g (global score), and F (faithfulness metrics) — four convergent signals for robust model debugging.
